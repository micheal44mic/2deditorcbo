PROMPT PER CODEX - SOLO MOTORE BRUSH SHAPE

Sto usando Codex con un LLM dentro al progetto. Devi lavorare come un senior WebGL2 graphics engineer. Non devi rifare la UI: la UI la faccio io. Tu devi solo far funzionare il motore in modo pulito, performante e scalabile.

Obiettivo engine-only
Implementare nel BrushEngine il comportamento reale di questi parametri della shape:

- shapeRotation
  - valore interno previsto in brushState: -1 -> +1
  - UI corrispondente: -100% -> +100%
  - 0 = la shape resta statica.
  - +1 = la shape segue la direzione del tratto.
  - -1 = la shape ruota in direzione inversa rispetto al tratto.

- shapeScatter
  - valore interno previsto in brushState: 0 -> 2
  - UI corrispondente: 0% -> 200%
  - randomizza la rotazione di ogni stamp.
  - non dipende dalla direzione del tratto.
  - 2 deve corrispondere a rotazione random completa.

- shapeCount
  - valore interno previsto in brushState: integer 1 -> 16
  - UI corrispondente: 1 -> 16
  - numero di shape stampate nello stesso punto.

- shapeCountJitter
  - valore interno previsto in brushState: 0 -> 1
  - UI corrispondente: 0% -> 100%
  - randomizza il count effettivo tra un minimo e shapeCount.
  - formula consigliata:
    minCount = max(1, ceil(shapeCount * (1 - shapeCountJitter)))
    effectiveCount = random integer tra minCount e shapeCount

1. Shader brush attuale in js/brush-engine.js

```js
const BRUSH_VERTEX_SHADER_SOURCE = `#version 300 es
precision highp float;

layout(location = 0) in vec2 a_position;
layout(location = 1) in vec2 aInstancePos;
layout(location = 2) in float aInstancePressure;
layout(location = 3) in float aInstanceAlpha;
layout(location = 4) in float aInstanceSizeScale;

uniform vec2 u_docResolution;
uniform float u_brushSize;
uniform float u_minSizeRatio;

out vec2 v_uv;
out float v_alpha;

void main() {
  float pressure = clamp(aInstancePressure, 0.0, 1.0);
  float sizeFactor = mix(u_minSizeRatio, 1.0, pressure);
  float scale = max(aInstanceSizeScale, 0.0);
  vec2 documentPosition = aInstancePos + a_position * u_brushSize * sizeFactor * scale;
  vec2 clipPosition = (documentPosition / u_docResolution) * 2.0 - 1.0;

  clipPosition.y *= -1.0;
  v_uv = a_position + 0.5;
  v_alpha = clamp(aInstanceAlpha, 0.0, 1.0);
  gl_Position = vec4(clipPosition, 0.0, 1.0);
}
`;

const BRUSH_FRAGMENT_SHADER_SOURCE = `#version 300 es
precision highp float;

uniform vec3 u_color;
uniform float u_flow;
uniform float u_hardness;
uniform sampler2D u_shapeTexture;
uniform float u_useShapeTexture;

in vec2 v_uv;
in float v_alpha;

out vec4 outColor;

void main() {
  float shape = 1.0;

  if (u_useShapeTexture > 0.5) {
    shape = texture(u_shapeTexture, v_uv).a;
  } else {
    float distanceFromCenter = distance(v_uv, vec2(0.5));

    if (distanceFromCenter > 0.5) {
      discard;
    }

    float fw = max(fwidth(distanceFromCenter), 0.001);
    float edgeStart = mix(0.0, 0.5 - fw, clamp(u_hardness, 0.0, 1.0));
    shape = 1.0 - smoothstep(edgeStart, 0.5, distanceFromCenter);
  }

  if (shape <= 0.001) {
    discard;
  }

  float alpha = shape * v_alpha * clamp(u_flow, 0.0, 1.0);
  outColor = vec4(u_color * alpha, alpha);
}
`;
```

Implementazione attesa nello shader:
- Aggiungi un attributo per la rotazione per istanza, ad esempio layout(location = 5) in float aInstanceRotation.
- Ruota a_position nel vertex shader prima di moltiplicarlo per brush size / pressure / scale.
- Mantieni v_uv = a_position + 0.5, cosi la texture shape resta campionata nello spazio locale del dab mentre il quad ruota nello spazio documento.

Schema suggerito:

```glsl
float angle = aInstanceRotation;
float c = cos(angle);
float s = sin(angle);
vec2 rotatedPosition = vec2(
  a_position.x * c - a_position.y * s,
  a_position.x * s + a_position.y * c
);
vec2 documentPosition = aInstancePos + rotatedPosition * u_brushSize * sizeFactor * scale;
```

Semantica rotation consigliata

```js
const directionAngle = Math.atan2(tangent.y, tangent.x);
const rotationFollow = this.clamp(this.brushState.shapeRotation, -1, 1);
const directionalRotation = directionAngle * rotationFollow;
```

Note:
- shapeRotation 0 => directionalRotation 0.
- shapeRotation 1 => segue directionAngle.
- shapeRotation -1 => inverse direction.
- Se il verso risulta visivamente invertito per via dell'asse Y DOM/WebGL, correggi in modo esplicito e commenta brevemente.

Semantica scatter consigliata

```js
const scatter = this.clamp(this.brushState.shapeScatter, 0, 2);
const scatterAngle = this.randomSigned() * Math.PI * scatter;
```

Con scatter 2, randomSigned * 2PI copre rotazione completa.

Semantica count jitter consigliata

```js
const count = this.clamp(Math.round(this.brushState.shapeCount), 1, 16);
const jitter = this.clamp01(this.brushState.shapeCountJitter);
const minCount = Math.max(1, Math.ceil(count * (1 - jitter)));
const effectiveCount = jitter <= 0
  ? count
  : minCount + Math.floor(this.nextRandom() * (count - minCount + 1));
```
