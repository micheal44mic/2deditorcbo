GUIDA TECNICA: GAUSSIAN BLUR NON DISTRUTTIVO SUL LAYER SELEZIONATO
====================================================================

Scopo del documento
-------------------

Questa guida e' scritta per una persona che vede l'app per la prima volta e
non puo' aprire direttamente il codice sorgente. L'obiettivo e' spiegare come
implementare la fase 1 del Gaussian Blur:

- il blur e' un effetto non distruttivo;
- l'effetto vive sul singolo layer scelto;
- non modifica i pixel originali del layer;
- undo/redo deve funzionare cambiando solo lo stato del layer;
- la parte UI non e' inclusa in questo documento.

La UI verra' collegata separatamente. Qui interessa solo il modello dati, il
renderer WebGL e i test necessari.


Com'e' fatta l'app, in breve
----------------------------

L'app e' un editor 2D in JavaScript vanilla. Non usa React/Vue. Il documento
visivo e' composto da layer. Ogni layer ha un record nello stato documento e,
se contiene pixel, una texture WebGL collegata al suo id.

I file chiave sono:

- js/document/document-layer-model.js
  Contiene il modello dei layer: creazione, ordine, visibilita', lock,
  opacity, active layer, gruppi e serializzazione dello stato.

- js/document/document-renderer.js
  Contiene il renderer WebGL2. Disegna i layer dal basso verso l'alto, gestisce
  texture e framebuffer, clipping mask, preview cache, trasformazioni raster e
  puppet transform.

- js/document/document-history.js
  Registra undo/redo. Le modifiche allo stato layer entrano nella history come
  "layer-state". Le modifiche pixel usano snapshot raster separati, ma per il
  blur fase 1 non servono snapshot pixel.

- js/layers-panel.js
  Sincronizza il pannello layer con il modello. Quando il DOM del pannello viene
  serializzato, conserva le proprieta' extra dei layer. Questo e' importante:
  una proprieta' nuova come effects resta attaccata al layer.

- js/text/vector-text-renderer.js
  I layer testo vettoriale vengono convertiti/cache-ati anche dentro una texture
  WebGL per partecipare al compositing normale. Quindi il Gaussian Blur del
  renderer puo' agire anche sui text layer quando la cache raster e' pronta.


Decisione architetturale
------------------------

Per la fase 1 NON creare un nuovo tipo di "adjustment layer".

Implementare invece un effetto live sul layer:

  layer.effects = [
    {
      type: "gaussian-blur",
      radius: 12,
      enabled: true
    }
  ]

Motivo:

- il modello layer supporta gia' proprieta' extra;
- la history dei layer registra gia' qualsiasi cambiamento allo stato;
- il renderer disegna gia' un layer alla volta;
- il blur non deve toccare i pixel originali;
- in futuro si potra' ancora aggiungere un vero adjustment layer globale.


Contratto dati consigliato
--------------------------

Ogni layer puo' avere un array effects. Per ora supportiamo solo:

  {
    type: "gaussian-blur",
    radius: Number,
    enabled: Boolean
  }

Regole:

- radius <= 0 significa nessun blur;
- enabled === false significa effetto ignorato;
- radius va clampato, per esempio tra 0 e 40 o tra 0 e 64;
- l'effetto va salvato sul layer, non sul renderer;
- la UI deve chiamare layerModel.updateLayer(...), ma la UI e' fuori scope.

Nota sulla forma dati:

Si potrebbe anche usare un oggetto tipo:

  effects: {
    gaussianBlur: { enabled: true, radius: 12 }
  }

Per questa guida si preferisce l'array, perche' in futuro permette piu' effetti
ordinati sullo stesso layer. L'importante e' scegliere una forma e restare
coerenti in modello, renderer e test.

Esempio di aggiornamento dati, senza UI:

  function setLayerGaussianBlur(layerModel, layerId, radius) {
    const layer = layerModel.findEntryById(layerId);

    if (!layer || layer.locked === true || layer.type === "background") {
      return false;
    }

    const nextRadius = Math.max(0, Math.min(40, Number(radius) || 0));
    const effects = Array.isArray(layer.effects)
      ? layer.effects.filter((effect) => effect && effect.type !== "gaussian-blur")
      : [];

    if (nextRadius > 0) {
      effects.push({
        type: "gaussian-blur",
        radius: nextRadius,
        enabled: true
      });
    }

    return layerModel.updateLayer(layerId, { effects }, {
      historyGroup: `gaussian-blur-${layerId}`,
      source: "gaussian-blur"
    });
  }

Nota: questo snippet mostra solo il contratto dati. Non e' la UI.


Normalizzazione nel modello layer
---------------------------------

Anche se il modello conserva gia' proprieta' extra, conviene aggiungere una
normalizzazione esplicita. Questo evita radius non numerici, effetti disabilitati
scritti male o forme dati parziali.

Metodo consigliato in DocumentLayerModel:

  normalizeLayerEffects(effects) {
    if (!Array.isArray(effects)) {
      return [];
    }

    return effects
      .filter(Boolean)
      .map((effect) => {
        if (effect.type !== "gaussian-blur") {
          return this.cloneValue(effect);
        }

        const radius = Number(effect.radius);

        return {
          type: "gaussian-blur",
          enabled: effect.enabled !== false,
          radius: Number.isFinite(radius)
            ? Math.max(0, Math.min(40, radius))
            : 0
        };
      })
      .filter((effect) => effect.type !== "gaussian-blur" || effect.radius > 0);
  }

Poi, in createBaseLayer(...), dopo la costruzione del layer:

  const layer = {
    ...extras,
    id: ...,
    type,
    name: ...,
    visible: ...,
    locked: ...,
    opacity: ...,
    clippingMask: ...
  };

  if (Array.isArray(options.effects)) {
    layer.effects = this.normalizeLayerEffects(options.effects);
  }

  return layer;


Perche' la history funziona quasi gratis
----------------------------------------

Il modello layer ha gia' updateLayer(id, patch, options). Quando updateLayer
viene chiamato, il modello:

- clona lo stato prima della modifica;
- applica il patch;
- emette un evento change;
- registra una entry "layer-state" nella history.

Poiche' effects e' una proprieta' del layer, undo/redo deve gia' ripristinarla
insieme a name, opacity, clippingMask, ecc.

Servono comunque test, per evitare regressioni.


Renderer: cosa bisogna aggiungere
---------------------------------

Il renderer oggi compone i layer cosi':

1. prende l'ordine dei layer dal modello;
2. parte dal basso;
3. per ogni layer trova la texture del layer;
4. applica opacity, clipping mask e preview stroke se presenti;
5. disegna la texture sul canvas finale.

Per aggiungere il blur non distruttivo:

1. prima di disegnare un layer, controllare se ha effects gaussian-blur;
2. se non ha blur, usare la texture originale;
3. se ha blur, renderizzare la texture originale dentro texture temporanee;
4. fare due passaggi WebGL:
   - passaggio orizzontale;
   - passaggio verticale;
5. disegnare il risultato blur sul canvas finale;
6. lasciare intatta la texture originale del layer.


Shader consigliato
------------------

Usare un blur separabile. Non fare un kernel 2D completo: costa troppo.

Passaggio 1:

  source layer texture -> scratch A
  u_texelStep = vec2(1.0 / width, 0.0)

Passaggio 2:

  scratch A -> scratch B
  u_texelStep = vec2(0.0, 1.0 / height)

Risultato:

  scratch B.texture

Fragment shader indicativo:

  #version 300 es
  precision highp float;

  uniform sampler2D u_texture;
  uniform vec2 u_texelStep;
  uniform float u_radius;

  in vec2 v_uv;
  out vec4 outColor;

  const int MAX_RADIUS = 40;

  void main() {
    float radius = clamp(u_radius, 0.0, float(MAX_RADIUS));

    if (radius <= 0.01) {
      outColor = texture(u_texture, v_uv);
      return;
    }

    float sigma = max(radius * 0.5, 0.5);
    float twoSigmaSq = 2.0 * sigma * sigma;
    vec4 sum = texture(u_texture, v_uv);
    float weightSum = 1.0;

    for (int i = 1; i <= MAX_RADIUS; i++) {
      if (float(i) > radius) {
        break;
      }

      float x = float(i);
      float weight = exp(-(x * x) / twoSigmaSq);
      vec2 offset = u_texelStep * x;

      sum += texture(u_texture, v_uv + offset) * weight;
      sum += texture(u_texture, v_uv - offset) * weight;
      weightSum += weight * 2.0;
    }

    outColor = sum / weightSum;
  }

Nota alpha:

Il renderer usa blending premoltiplicato:

  gl.blendFunc(gl.ONE, gl.ONE_MINUS_SRC_ALPHA)

Quindi e' corretto blurrare colori gia' premoltiplicati. Non dividere per alpha
nello shader, almeno per la prima versione.


Risorse WebGL da aggiungere
---------------------------

Nel renderer servono:

- blurProgramInfo
  Programma shader del blur, con uniform:
  texture, texelStep, radius.

- blurScratchA
  Texture + framebuffer temporanei a dimensione documento.

- blurScratchB
  Texture + framebuffer temporanei a dimensione documento.

Non inserire questi target temporanei in rasterTargetsByLayerId. Quella mappa
contiene contenuto reale dei layer. Le scratch texture sono risorse interne del
renderer e devono vivere fuori dalla mappa.

La dimensione deve essere la stessa del documento, non del viewport:

  width = renderer.width
  height = renderer.height

Usare CLAMP_TO_EDGE su S e T. Per i filtri, LINEAR o NEAREST possono funzionare;
per coerenza con il documento va bene partire con LINEAR sui target temporanei.

Importante: queste risorse vanno eliminate in dispose().


API renderer consigliate
------------------------

Aggiungere funzioni private/metodi sulla classe DocumentRenderer.

1. getGaussianBlurRadius(layer)

  getGaussianBlurRadius(layer) {
    const effects = Array.isArray(layer?.effects) ? layer.effects : [];
    const effect = effects.find((item) =>
      item &&
      item.type === "gaussian-blur" &&
      item.enabled !== false
    );
    const radius = Number(effect?.radius);

    return Number.isFinite(radius)
      ? Math.max(0, Math.min(40, radius))
      : 0;
  }

2. hasEnabledLayerEffects(layer)

  hasEnabledLayerEffects(layer) {
    return this.getGaussianBlurRadius(layer) > 0;
  }

3. hasAnyEnabledLayerEffects(layers)

  hasAnyEnabledLayerEffects(layers = this.getOrderedLayersBottomToTop()) {
    return Array.isArray(layers) &&
      layers.some((layer) => this.hasEnabledLayerEffects(layer));
  }

4. ensureBlurResources()

Deve creare programma shader e due target scratch se mancano.

5. applyGaussianBlurTexture(sourceTexture, radius, options)

Input:

  sourceTexture
  radius
  width
  height

Output:

  texture blur oppure sourceTexture se radius <= 0

Pseudo-codice:

  applyGaussianBlurTexture(sourceTexture, radius, options = {}) {
    if (!sourceTexture || radius <= 0) {
      return sourceTexture;
    }

    const resources = this.ensureBlurResources();
    const width = options.width || this.width;
    const height = options.height || this.height;

    this.runBlurPass({
      sourceTexture,
      target: resources.scratchA,
      radius,
      texelStepX: 1 / width,
      texelStepY: 0
    });

    this.runBlurPass({
      sourceTexture: resources.scratchA.texture,
      target: resources.scratchB,
      radius,
      texelStepX: 0,
      texelStepY: 1 / height
    });

    return resources.scratchB.texture;
  }

6. getLayerRenderTexture(layer, layerTarget)

  getLayerRenderTexture(layer, layerTarget) {
    if (!layerTarget?.texture) {
      return null;
    }

    const radius = this.getGaussianBlurRadius(layer);

    if (radius <= 0) {
      return layerTarget.texture;
    }

    return this.applyGaussianBlurTexture(layerTarget.texture, radius, {
      height: layerTarget.height || this.height,
      width: layerTarget.width || this.width
    });
  }


Punto di integrazione principale: drawToCanvas
----------------------------------------------

Nel ciclo che disegna i layer, oggi la logica equivalente e':

  for (const layer of orderedLayers) {
    const layerTarget = this.rasterTargetsByLayerId.get(layer.id);
    const opacity = ...

    if (layerTarget?.texture) {
      drawTexture(layerTarget.texture, opacity, null, clipBase);
    }
  }

Va trasformata concettualmente cosi':

  for (const layer of orderedLayers) {
    const layerTarget = this.rasterTargetsByLayerId.get(layer.id);
    const opacity = ...
    const renderTexture = this.getLayerRenderTexture(layer, layerTarget);

    if (renderTexture) {
      drawTexture(renderTexture, opacity, null, clipBase);
    }
  }

Subito dopo un blur pass bisogna tornare al programma standard del renderer,
perche' il blur usa un altro shader:

  const renderTexture = this.getLayerRenderTexture(layer, layerTarget);
  bindArtboardProgram();
  drawTexture(renderTexture, opacity, null, clipBase);

Questa chiamata a bindArtboardProgram() e' importante.


Preview cache: cosa fare subito
-------------------------------

Il renderer ha una preview cache mipmapped per lo zoom-out. Se non la tocchi,
rischi questo bug:

- a zoom normale il layer e' blur;
- a zoom-out il renderer usa la cache vecchia/non filtrata;
- il blur sparisce o resta incoerente.

La soluzione piu' completa e' far usare anche a updatePreviewCache() lo stesso
percorso di risoluzione texture usato da drawToCanvas(), cioe':

  raw layer texture -> effects -> compositing

Se non lo fai subito, la soluzione minima corretta e' disabilitare la preview
cache quando esiste almeno un effetto attivo:

  const hasLayerEffects = this.hasAnyEnabledLayerEffects(orderedLayers);

  const canUsePreviewCache = Boolean(
    isZoomedOut &&
    !hasLayerEffects &&
    ...
  );

Cosi' quando c'e' un blur attivo il renderer usa il percorso full-res e resta
corretto.

Attenzione live stroke:

Se l'utente sta disegnando sul layer blur, una stroke live non deve apparire
nitida sopra una cache blur. Per la prima versione, disabilitare preview cache
anche quando:

- c'e' activeStrokeTexture;
- il layer attivo ha gaussian-blur;
- la stroke non e' stata compositata dentro la texture scratch prima del blur.


Puppet transform: rischio noto
------------------------------

Il renderer ha una strada speciale per i layer con puppet pins:

  hasPuppetLayerTransform(layer)
  drawPuppetLayer(...)

La versione iniziale puo' scegliere una di queste due strategie:

A. Supporto semplice:
   applica il blur alla texture sorgente e poi passa la texture blur a
   drawPuppetLayer come sourceTexture.

B. Supporto perfetto:
   renderizza prima il puppet deformato dentro un target temporaneo, poi blurra
   quel risultato.

Per la fase 1 e' accettabile la strategia A, perche' conserva l'effetto live e
non modifica i pixel. La differenza visiva si nota solo se la deformazione e'
molto forte.

Pseudo-codice per strategia A:

  const renderTexture = this.getLayerRenderTexture(layer, layerTarget);

  const didDrawPuppet = this.drawPuppetLayer(layer, layerTarget, opacity, {
    camera,
    sourceTexture: renderTexture,
    viewportHeight,
    viewportWidth
  });


Clipping mask: rischio noto
---------------------------

Le clipping mask usano l'alpha del layer sotto come base. Se il layer base ha
blur, ci sono due interpretazioni possibili:

- usare l'alpha originale come base della clipping mask;
- usare l'alpha blur come base.

Per la fase 1, tenere l'alpha originale e' piu' semplice e stabile. Quindi:

- disegnare il base layer blur;
- continuare a usare layerTarget.texture originale per clipBase.

Se in futuro si vuole che la clipping mask segua anche il bordo blur, servira'
una gestione piu' attenta delle scratch texture per non sovrascriverle mentre
vengono ancora usate come clip texture.


Raster transform preview: rischio noto
--------------------------------------

Quando un layer e' in trasformazione raster, il renderer:

- taglia/maschera temporaneamente la sorgente;
- disegna una preview trasformata sopra.

Per la fase 1, evitare di far passare la preview trasformata dentro lo stesso
blur pipeline, a meno che non si voglia gestire un target temporaneo separato.

Comportamento accettabile iniziale:

- il layer normale viene blur;
- durante la transform preview, la preview puo' essere non blur o blur pre-
  transform;
- dopo commit, il layer torna a renderizzare con blur normale.


Text layer: cosa aspettarsi
---------------------------

I text layer vettoriali hanno una cache raster nel renderer. Il blur applicato
nel compositing WebGL puo' quindi funzionare anche sul testo.

Durante editing/interazione, l'overlay SVG del testo puo' mostrare temporaneamente
un'anteprima non blur. Non e' un bug critico per fase 1: il documento composito
finale deve essere blur.

Non aggiungere effects alla signature della cache testo. Cambiare radius blur
non deve obbligare il text renderer a rasterizzare di nuovo il testo: il blur
succede dopo, nel renderer documento.

Se in futuro si usa "rasterizza testo" e si vuole preservare esattamente
l'aspetto, copiare effects dal text layer al nuovo paint layer rasterizzato.


Cleanup in dispose()
--------------------

Quando il renderer viene distrutto, eliminare:

- blurProgramInfo.program;
- framebuffer scratch A/B;
- texture scratch A/B.

Esempio concettuale:

  deleteBlurResources() {
    const gl = this.gl;

    if (this.blurScratchA?.framebuffer) {
      gl.deleteFramebuffer(this.blurScratchA.framebuffer);
    }

    if (this.blurScratchA?.texture) {
      gl.deleteTexture(this.blurScratchA.texture);
    }

    if (this.blurScratchB?.framebuffer) {
      gl.deleteFramebuffer(this.blurScratchB.framebuffer);
    }

    if (this.blurScratchB?.texture) {
      gl.deleteTexture(this.blurScratchB.texture);
    }

    if (this.blurProgramInfo?.program) {
      gl.deleteProgram(this.blurProgramInfo.program);
    }

    this.blurScratchA = null;
    this.blurScratchB = null;
    this.blurProgramInfo = null;
  }

Chiamarla dentro dispose().

Definizione di "finito" per fase 1
----------------------------------

La fase 1 e' completa quando:

- un layer puo' avere effects gaussian-blur nello stato;
- cambiando radius il canvas si aggiorna senza modificare i pixel originali;
- undo/redo ripristina radius e enabled;
- nascondere/mostrare il layer continua a funzionare;
- opacity del layer continua a funzionare sopra il blur;
- il blur non sparisce a zoom-out;
- nessuna texture/framebuffer blur resta viva dopo dispose;
- i test Node passano.


Fuori scope
-----------

Queste cose NON fanno parte della fase 1:

- creare un adjustment layer globale;
- blurrare tutto cio' che sta sotto un layer;
- applicare permanentemente il blur ai pixel;
- UI, pannelli, slider, menu, icone e testi visibili;
- esportazione file;
- ottimizzazioni avanzate della preview cache.
