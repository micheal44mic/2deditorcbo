GUIDA IMPLEMENTAZIONE CLIPPING MASK - EDITOR 2D CBO
====================================================

Scopo del documento
-------------------

Questo documento spiega come implementare la CLIPPING MASK in questa app per una
persona che vede il progetto per la prima volta e non puo' aprire i sorgenti.

Non parla di layer mask dipinte. Qui parliamo della logica tipo Photoshop:
un layer sopra viene mostrato solo dentro l'alpha del layer sotto.

Esempio visivo:

  Texture colore      [clipping mask ON]
  Ombra               [clipping mask ON]
  Forma base          [normale]
  Background          [normale]

Texture colore e Ombra restano layer indipendenti, ma si vedono solo dove
Forma base ha pixel visibili. I pixel fuori dalla forma NON vengono cancellati:
sono solo nascosti dal compositing.


Glossario minimo
----------------

Layer base:
  Il primo layer valido sotto a uno o piu' clipping layer. La sua trasparenza
  decide dove i layer clippati si vedono.

Clipping layer:
  Un layer con flag "clippingMask: true". Si disegna normalmente nella sua
  texture, ma in output viene moltiplicato per l'alpha del layer base.

Alpha:
  Canale trasparenza di una texture. 0 = invisibile, 1 = visibile.

FBO:
  Framebuffer WebGL. In questa app ogni layer raster ha una texture/FBO.

Premultiplied alpha:
  Questa app usa colori premoltiplicati. Se si applica una clipping mask, bisogna
  moltiplicare tutto il vec4 colore, non solo outColor.a.


Comportamento richiesto
-----------------------

1. L'utente fa tasto destro su un layer.
2. Nel menu contestuale sceglie "CLIPPING MASK".
3. Il layer selezionato riceve il flag:

     clippingMask: true

4. La UI del layer mostra un indicatore/freccia a sinistra della riga, cosi'
   si capisce che quel layer e' clippato al layer sotto.
5. Il renderer WebGL disegna quel layer usando l'alpha del layer base sotto.
6. Se piu' layer consecutivi sopra hanno clippingMask true, tutti usano lo
   stesso layer base sotto.
7. Se il clipping viene disattivato, i pixel tornano visibili normalmente.
8. Undo/redo deve funzionare per il toggle del flag, usando la history layer
   gia' esistente.


Regole consigliate per V1
-------------------------

Un layer puo' diventare clipping mask solo se:

  - non e' un gruppo
  - non e' il background
  - non e' locked
  - sotto di lui esiste un layer contenuto valido

Un layer base valido e':

  - paint
  - image
  - vector-text se ha raster target sincronizzato

Per V1 e' accettabile non supportare perfettamente i confini dei gruppi:
l'app attuale renderizza i gruppi appiattendo i layer. Se si vuole una logica
identica a Photoshop dentro/fuori gruppo, serve un pass successivo.

Come sono ordinati i layer oggi
-------------------------------

Nel model, la lista entries e' top-to-bottom:

  entries[0] = layer piu' in alto
  entries[entries.length - 1] = background in basso

Nel renderer, invece, si disegna bottom-to-top.

Codice attuale da conoscere:

  // js/document/document-layer-model.js

  flattenTopToBottom(entries = this.entries, ancestorsVisible = true) {
    const result = [];

    for (const entry of entries) {
      const visible = ancestorsVisible && entry.visible !== false;

      if (entry.type === "group") {
        result.push(...this.flattenTopToBottom(entry.children || [], visible));
        continue;
      }

      result.push({
        ...this.cloneEntry(entry),
        visible,
      });
    }

    return result;
  }

  getRenderableLayers() {
    return this.flattenTopToBottom()
      .filter((entry) => entry.visible !== false)
      .reverse();
  }

Nota:
  getRenderableLayers() elimina i layer nascosti e poi inverte l'ordine.
  Per clipping mask sarebbe meglio avere anche una lista bottom-to-top che
  includa i layer nascosti, per evitare un bug:

    se il base layer e' hidden, i clipped layer sopra NON devono saltare al
    layer visibile ancora piu' sotto.


Metadata layer: cosa aggiungere
-------------------------------

Il modo piu' semplice e coerente e' aggiungere un campo booleano:

  clippingMask: true

Esempio layer:

  {
    id: "paint-abc",
    type: "paint",
    name: "Texture",
    visible: true,
    locked: false,
    opacity: 1,
    clippingMask: true
  }

Per fortuna il modello attuale conserva gia' i campi extra nei layer normali.
Questo e' il pezzo importante:

  function getLayerBelow(layerId) {
    const layers = getFlatContentLayersTopToBottom();
    const index = layers.findIndex((layer) => layer.id === layerId);

    return index >= 0 ? layers[index + 1] || null : null;
  }

  function isClippingMaskAllowed(layerId) {
    const layer = layerModel?.findEntryById?.(layerId);
    const below = getLayerBelow(layerId);

    if (!layer || layer.locked === true) {
      return false;
    }

    if (layer.type === "group" || layer.type === "background" || layer.id === "background") {
      return false;
    }

    if (!below || below.type === "background" || below.type === "group") {
      return false;
    }

    return true;
  }

  function toggleClippingMask(layerId) {
    const layer = layerModel?.findEntryById?.(layerId);

    if (!layer || !isClippingMaskAllowed(layerId)) {
      return false;
    }

    const nextValue = layer.clippingMask !== true;

    return layerModel.updateLayer(layerId, {
      clippingMask: nextValue,
    }, {
      historyGroup: `clipping-mask-${layerId}`,
      source: "layers-panel-clipping-mask",
    });
  }

Serializzazione layer panel
---------------------------

Il layer panel serializza il DOM nel model. Per fortuna preserva gia' i campi
extra trovati nel model:

  // js/layers-panel.js

  const preservedEntry = existingEntry
    ? Object.fromEntries(
        Object.entries(existingEntry)
          .filter(([key]) => !["id", "type", "name", "visible", "locked", "opacity", "children"].includes(key)),
      )
    : {};

  const serialized = {
    ...preservedEntry,
    id,
    type,
    name: ...,
    visible: ...,
    locked: ...,
    opacity: 1,
  };

Quindi clippingMask resta preservato durante rename, reorder, lock, visibility.
Da testare bene quando si copia un layer: il clone probabilmente copia anche
clippingMask. Questo e' corretto solo se il clone resta sopra un layer base
valido. Se si vuole evitare casi strani, durante clone si puo' azzerare:

  copiedLayer.clippingMask = false;

Ma per V1 si puo' anche preservare, perche' e' comportamento simile agli editor
grafici: duplicare un clipping layer mantiene il clipping.

-------------------------------

Il renderer oggi disegna ogni layer raster bottom-to-top.

Codice attuale importante:

        : null;

      if (layerTarget?.texture) {
        ...
        drawTexture(layerTarget.texture, opacity);
        ...
      }

La clipping mask va inserita qui.

Concetto:

  currentClipBase = ultimo layer non clippato incontrato sotto

Durante il loop bottom-to-top:

  - se layer.clippingMask !== true:
      diventa il nuovo currentClipBase
      si disegna normale

  - se layer.clippingMask === true:
      NON diventa base
      si disegna moltiplicato per alpha di currentClipBase

Esempio bottom-to-top:

  Background       normale, base potenziale ma da ignorare o considerare invalido
  Forma base       normale -> currentClipBase = Forma base
  Ombra            clipping -> usa alpha Forma base
  Texture          clipping -> usa alpha Forma base
  Lineart          normale -> currentClipBase = Lineart
  Highlight        clipping -> usa alpha Lineart

  uniform float u_previewCutMode;
  uniform vec4 u_previewCutRect;

  ...

  vec4 color = texture(u_texture, v_uv) * u_opacity;

  if (u_previewCutMode > 0.5) {
    ...
  }

  if (u_maskMode > 0.5) {
    float eraseAlpha = 0.0;

    if (u_maskRectMode > 0.5) {
      vec2 local = (v_documentPixel - u_maskRect.xy) / max(u_maskRect.zw, vec2(1.0));

      if (!any(lessThan(local, vec2(0.0))) && !any(greaterThan(local, vec2(1.0)))) {
        eraseAlpha = clamp(texture(u_maskTexture, vec2(local.x, 1.0 - local.y)).a, 0.0, 1.0);
      }
    } else {
      eraseAlpha = clamp(texture(u_maskTexture, v_uv).a, 0.0, 1.0);
    }

    color *= 1.0 - eraseAlpha;
  }

  outColor = color;

Non riusare u_maskMode per clipping mask, perche' quella logica e' "erase":

  color *= 1.0 - eraseAlpha

La clipping invece deve essere:

  color *= clipAlpha

Quindi servono uniform separati.


Shader renderer: codice da aggiungere
-------------------------------------

Aggiungere uniform:

  uniform sampler2D u_clipTexture;
  uniform float u_clipMode;
  uniform float u_clipOpacity;
  uniform vec2 u_clipTextureSize;
  uniform vec2 u_drawOrigin;

Perche' u_drawOrigin serve:
  Normalmente i layer sono texture grandi quanto il documento e v_uv basta.
  Ma lo stroke attivo del brush viene disegnato come texture piccola/ritagliata
  con activeStrokeRect. In quel caso v_documentPixel e' locale al rettangolo,
  quindi serve ricostruire la posizione reale nel documento:

    globalDocPixel = u_drawOrigin + v_documentPixel

Codice shader suggerito:

  if (u_clipMode > 0.5) {
    vec2 globalDocPixel = u_drawOrigin + v_documentPixel;
    vec2 clipUv = vec2(
      globalDocPixel.x / max(u_clipTextureSize.x, 1.0),
      1.0 - globalDocPixel.y / max(u_clipTextureSize.y, 1.0)
    );
    float clipAlpha = 0.0;

    if (
      clipUv.x >= 0.0 &&
      clipUv.x <= 1.0 &&
      clipUv.y >= 0.0 &&
      clipUv.y <= 1.0
    ) {
      clipAlpha = texture(u_clipTexture, clipUv).a * clamp(u_clipOpacity, 0.0, 1.0);
    }

    color *= clipAlpha;
  }

Posizione nel fragment shader:
  Applicare la clipping DOPO eventuale previewCut e DOPO eraser preview.
  Cosi':

    1. prendo colore layer
    2. applico preview cut transform se serve
    3. applico eraser preview se sto cancellando
    4. applico clipping mask
    5. outColor = color

    if (clipBase?.target?.texture) {
      gl.uniform1f(uniforms.clipMode, 0.0);
      gl.activeTexture(gl.TEXTURE2);
      gl.bindTexture(gl.TEXTURE_2D, null);
      gl.activeTexture(gl.TEXTURE0);
    }
  };

In bindArtboardProgram inizializzare i nuovi uniform:

  gl.uniform1f(uniforms.clipOpacity, 1.0);

Puppet e raster transform
-------------------------

Ci sono due percorsi speciali:

  drawPuppetLayer()
  drawPerspectiveTexturedQuad() / drawTexturedQuad()

Questi hanno shader separati. Se un layer con puppet transform e' clippingMask,
la clipping non viene applicata automaticamente dallo shader principale.

Per V1, scelta consigliata:

  - se layer.clippingMask true, disegnare il layer con drawTexture normale
    senza puppet transform, oppure disabilitare clipping per puppet layer.

Ma se si vuole supporto completo:

  - aggiungere u_clipTexture anche agli shader puppet/perspective
  - passare posizione documento o UV coerenti
  - moltiplicare outColor per clipAlpha

Questo e' piu' lungo. Non farlo nel primo step se l'obiettivo e' avere clipping
mask stabile sui layer raster normali.

Smudge
------

Smudge oggi scrive solo sui layer paint.

Codice attuale importante:

  // js/smudge-engine.js

  getActivePaintLayerTarget() {
    const layerModel = this.documentRenderer?.layerModel;

    }

    ...

Esattamente come il Brush Engine, lo Smudge altera i veri pixel della texture/FBO
del layer attivo. Poiche' la clipping mask in questa architettura e' un effetto
calcolato solo nel compositing visivo del renderer WebGL, lo Smudge non richiede
modifiche matematiche interne:

  - continua a leggere, spostare e sfumare i pixel reali sulla texture originale
    del layer clippato
  - i pixel possono "sbavare" anche fuori dall'alpha del layer base sottostante,
    ma non vengono mostrati a schermo mentre clippingMask e' attiva
  - se l'utente disattiva la clipping mask, vedra' l'intera sbavatura originale
    anche oltre i bordi del layer base

Questo e' il comportamento non distruttivo corretto.

Nota importante:
  Non dare per certo che Smudge usi lo stesso activeStrokeTexture del Brush
  Engine. Dal codice attuale lo Smudge ha una sua pipeline di scratch/history e
  poi chiede redraw. La regola resta comunque semplice: se il risultato finale
  passa da drawToCanvas(), e drawToCanvas() applica clipBase al layer, anche lo
  Smudge viene visualizzato correttamente senza cambiare la sua logica interna.

Merge Down e appiattimento
--------------------------

Se l'app prevede una funzione "Merge Down", bisogna trattare i clipping layer
con attenzione.

Unire un clipping layer sul suo layer base trasforma una modifica non distruttiva
in una modifica distruttiva. In quel caso i pixel del clipping layer fuori
dall'alpha del layer base dovrebbero essere eliminati in modo permanente durante
il merge.

Soluzione consigliata per V1:

  disabilitare Merge Down se il layer selezionato ha clippingMask === true.

Motivo:
  evitare di scrivere subito uno shader o un pass CPU dedicato al bake del
  clipping dentro il framebuffer di merge.

Soluzione V2:

  implementare un pass di bake:

    mergedColor = baseColor + clippedColor * baseAlpha

  rispettando premultiplied alpha, opacity layer, eventuali transform e history.


History / undo redo
-------------------

Il toggle clippingMask e' un cambio metadata del layer, non pixel.
Deve quindi usare la history layer-state gia' esistente.

DocumentLayerModel.updateLayer() registra history automaticamente se non viene
passato history:false.

Codice attuale importante:

  updateLayer(id, patch, options = {}) {
    const entry = this.findEntryById(id);

    if (!entry || entry.type === "group") {
      return false;
    }

    const nextPatch = typeof patch === "function" ? patch(this.cloneEntry(entry)) : patch;

    if (!nextPatch || typeof nextPatch !== "object") {
      return false;
    }

    const beforeState = this.captureHistoryState(options);

    Object.assign(entry, this.cloneValue(nextPatch));
    entry.opacity = Number.isFinite(entry.opacity) ? Math.min(1, Math.max(0, entry.opacity)) : 1;

    this.emitChange(options.source || "update-layer");
    this.recordHistoryStateChange(beforeState, options);

    return true;
  }

Decisioni V1 ed edge case
-------------------------

1. Background come base:
   Decisione V1: non usare background come base valida. E' opaco e renderebbe
   la clipping mask inutile.

2. Layer base nascosto:
   Decisione V1: il clipping layer deve sparire, non deve clippare sul layer
   piu' sotto. Per questo il renderer deve conoscere anche i layer hidden
   durante il calcolo currentClipBase.

3. Layer base locked:
   Locked non importa per il rendering. Un layer locked puo' essere base.

4. Layer clipping locked:
   Decisione V1: se locked, non permettere toggle da UI.

5. Group:
   Decisione V1: seguire il flatten attuale.
   V2: rispettare confini gruppo.

6. Opacity del base:
   Decisione V1: clipAlpha = baseTextureAlpha * baseLayerOpacity.
   Se si vuole comportamento piu' simile a certi editor, si puo' usare solo
   baseTextureAlpha. La scelta deve essere testata visivamente.

7. Opacity del clipping layer:
   Continua ad applicarsi normalmente:

     color = layerTexture * clippingLayerOpacity * baseAlpha

8. Preview cache:
   Decisione V1: disabilitare preview cache quando almeno un layer ha
   clippingMask true. E' meno performante a zoom out, ma evita render errati.

9. Puppet/transform su layer clippati:
   Decisione V1: non inseguire il supporto completo negli shader puppet e
   perspective. Bloccare o limitare i casi ambigui, poi implementare supporto
   completo in V2.

10. Indicatore UI:
   Decisione V1: la freccia/indicatore sta sul layer clippato, a sinistra della
   sua riga. Non sta sul layer base.

      gl.uniform2f(uniforms.drawOrigin, 0, 0);
    }

      const clipOpacity = Number.isFinite(clipBase.layer?.opacity)
        ? Math.min(1, Math.max(0, clipBase.layer.opacity))
        : 1;

      gl.activeTexture(gl.TEXTURE2);
      gl.bindTexture(gl.TEXTURE_2D, clipBase.target.texture);
      gl.uniform1i(uniforms.clipTexture, 2);
      gl.uniform1f(uniforms.clipMode, 1.0);
      gl.uniform1f(uniforms.clipOpacity, clipOpacity);

        clipBase.target.height || target.height,

      gl.uniform1f(uniforms.clipOpacity, 1.0);
      gl.uniform2f(uniforms.clipTextureSize, target.width, target.height);

    gl.uniform1f(uniforms.opacity, opacity);
    gl.drawArrays(gl.TRIANGLE_STRIP, 0, 4);

Definizione finale
------------------

La clipping mask in questa app deve essere un flag di compositing:

  layer.clippingMask === true

Non deve:

  - creare una texture mask separata
  - cancellare pixel
  - modificare il layer base
  - fare bake automatico

Deve:

  - usare l'alpha del layer sotto come maschera
  - essere reversibile
  - funzionare con undo/redo
  - essere visibile nella UI layer con freccia a sinistra
  - essere attivabile/disattivabile da tasto destro sul layer


Fine della guida implementativa per CLIPPING MASK V1.
